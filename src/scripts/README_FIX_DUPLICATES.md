# Исправление дублирующихся ID участников

## Проблема

В базе данных обнаружены дублирующиеся `participantId` из-за race condition в функции генерации ID. Это происходило, когда несколько пользователей регистрировались одновременно.

## Решение

### 1. Исправлена функция генерации ID

Обновлена функция `generateNextParticipantId()` в файле `src/lib/dataService.js`:
- Теперь использует Firebase Transaction для атомарных операций
- Создаёт счётчик в коллекции `counters` для отслеживания последнего номера
- Гарантирует уникальность ID даже при одновременных регистрациях

### 2. Скрипт для исправления существующих дубликатов

Создан скрипт `fixDuplicateParticipantIds.cjs`, который:
- Находит все дублирующиеся ID
- Оставляет первый документ с оригинальным ID (по дате создания)
- Переназначает новые уникальные ID остальным дубликатам
- Присваивает ID документам без `participantId`
- Создаёт/обновляет счётчик для будущих регистраций

## Инструкция по запуску

### Шаг 1: Убедитесь, что .env файл настроен

Проверьте наличие файла `.env.local` в корне проекта с Firebase конфигурацией:

```env
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_auth_domain
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_storage_bucket
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id
NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID=your_measurement_id
```

### Шаг 2: Запустите скрипт

```bash
# Из корневой директории проекта
node src/scripts/fixDuplicateParticipantIds.cjs
```

### Шаг 3: Проверьте результаты

Скрипт выведет подробную информацию о:
- Количестве найденных дубликатов
- Обновлённых документах
- Следующем доступном номере для новых регистраций

## Что делает скрипт

1. **Анализ базы данных:**
   - Загружает все регистрации
   - Группирует по `participantId`
   - Находит дубликаты

2. **Исправление дубликатов:**
   - Для каждой группы дубликатов оставляет первый документ (по дате)
   - Остальным присваивает новые уникальные ID

3. **Обработка документов без ID:**
   - Присваивает новые ID документам, у которых `participantId` отсутствует или равен "—"

4. **Создание счётчика:**
   - Создаёт документ счётчика в коллекции `counters`
   - Устанавливает текущий номер для корректной работы будущих регистраций

## Важно

- **Создайте резервную копию** базы данных перед запуском (через Firebase Console)
- Скрипт безопасен и не удаляет данные, только обновляет `participantId`
- После исправления все новые регистрации будут получать уникальные ID

## Проверка после исправления

После запуска скрипта проверьте:
1. Админ-панель: `/admin/participants` - убедитесь, что все ID уникальны
2. Попробуйте зарегистрировать нового участника - должен получить следующий порядковый номер
3. Проверьте наличие документа счётчика в коллекции `counters`

## Troubleshooting

**Ошибка "Firebase not initialized":**
- Проверьте правильность переменных окружения в `.env.local`

**Ошибка "Permission denied":**
- Убедитесь, что у вас есть права администратора Firebase
- Проверьте правила безопасности Firestore

**Скрипт не находит дубликаты:**
- Возможно, дубликаты уже были исправлены
- Проверьте вывод: "✅ Дубликаты не найдены!"
