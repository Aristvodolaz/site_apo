# 🚨 Решение проблемы блокировки писем как спам

## Проблема

При массовой рассылке Yandex блокирует письма с ошибкой:
```
554 5.7.1 Message rejected under suspicion of SPAM
```

## Причины блокировки

1. **Слишком много писем за короткое время** - Yandex ограничивает количество писем
2. **Одинаковое содержимое** - все письма имеют одинаковый текст
3. **Отсутствие SPF/DKIM/DMARC** - домен не настроен для отправки почты
4. **Низкая репутация отправителя** - новый или неиспользуемый аккаунт

## ✅ Решения

### 1. Увеличить задержку между письмами

**Текущая задержка:** 5 секунд  
**Рекомендуемая:** 10-15 секунд

Скрипт уже обновлен с задержкой 10 секунд.

### 2. Разбить рассылку на части

Вместо отправки всех писем сразу, разбить на партии:

```bash
# Первая партия: записи 0-1000
node src/scripts/sendEmailBySubject.cjs biology 0

# Подождать 1-2 часа, затем вторая партия: записи 1000-2000
node src/scripts/sendEmailBySubject.cjs biology 1000

# И так далее...
```

### 3. Настроить SPF/DKIM для домена

Если вы используете домен `arctolymp.ru`, нужно настроить DNS записи:

**SPF запись:**
```
TXT @ "v=spf1 include:_spf.yandex.ru ~all"
```

**DKIM:** Настраивается в панели Yandex для домена

**DMARC:**
```
TXT _dmarc "v=DMARC1; p=none; rua=mailto:admin@arctolymp.ru"
```

### 4. Использовать другой SMTP сервис

Для массовых рассылок лучше использовать специализированные сервисы:

- **SendGrid** - до 100 писем/день бесплатно
- **Mailgun** - до 5000 писем/месяц бесплатно
- **Amazon SES** - очень дешево для больших объемов
- **Postmark** - для транзакционных писем

### 5. Вариативность в письмах

Добавить небольшие вариации в текст письма (например, разные приветствия).

## 🔧 Что делать сейчас

### Вариант 1: Продолжить с увеличенной задержкой

Скрипт уже обновлен с задержкой 10 секунд. При ошибках спама автоматически добавляется дополнительная задержка 15 секунд.

### Вариант 2: Разбить на партии

```bash
# Партия 1: первые 500 участников
node src/scripts/sendEmailBySubject.cjs biology 0

# Подождать 1-2 часа

# Партия 2: следующие 500 участников
node src/scripts/sendEmailBySubject.cjs biology 500

# И так далее...
```

### Вариант 3: Повторная отправка только заблокированных

Создать скрипт, который повторно отправит письма только тем, кто получил ошибку спама (через несколько часов).

## 📊 Статистика ошибок

Из ваших логов видно:
- ✅ Много успешных отправок
- ❌ Много ошибок "SPAM" (начиная примерно с 50% рассылки)
- ⚠️ Это означает, что Yandex начал блокировать после определенного количества писем

## 💡 Рекомендации

1. **Немедленно:** Разбейте рассылку на партии по 300-500 писем с перерывами 1-2 часа
2. **Краткосрочно:** Увеличьте задержку до 15 секунд
3. **Долгосрочно:** Настройте SPF/DKIM для домена или используйте специализированный сервис

## 🔄 Повторная отправка заблокированных

После завершения рассылки можно создать скрипт для повторной отправки только тем, кто получил ошибку спама (через 24 часа).

---

**Важно:** Yandex может временно заблокировать аккаунт при слишком частых рассылках. Рекомендуется делать перерывы между партиями.
