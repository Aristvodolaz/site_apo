# 🚀 Быстрый старт: Email-рассылка участникам

## 📝 Кратко

Отправка писем участникам с напоминанием о номере регистрации и информацией о Telegram боте @ArctolympBot.

## ⚡ Быстрые команды

```bash
# 1. СНАЧАЛА ТЕСТ (отправка на kap.moral@mail.ru)
npm run send-test-email

# 2. ПОСЛЕ ТЕСТА - массовая рассылка (на все email из Firebase)
npm run send-mass-email

# 3. ПОВТОРНАЯ ОТПРАВКА (если были ошибки "Too many messages")
npm run retry-failed-emails
```

## ✅ Что нужно проверить перед запуском:

1. **Файл .env содержит:**
   ```
   SMTP_HOST=smtp.yandex.ru
   SMTP_PORT=465
   SMTP_SECURE=true
   SMTP_USER=ваш_email@yandex.ru
   SMTP_PASSWORD=ваш_пароль_приложения
   SMTP_FROM="Арктическая олимпиада <ваш_email@yandex.ru>"
   ```

2. **В Firebase коллекции `registrations` есть записи с:**
   - `email` - почта участника
   - `participantId` - номер типа APO-2026-00001
   - `firstName` - имя участника

## 🧪 Шаг 1: Тестовая отправка

```bash
npm run send-test-email
```

**Что произойдет:**
- Возьмет данные ПЕРВОГО участника из Firebase
- Отправит письмо на **kap.moral@mail.ru**
- Покажет детальный лог

**Что проверить:**
- ✅ Письмо пришло на kap.moral@mail.ru
- ✅ Оформление выглядит красиво
- ✅ Номер участника отображается правильно
- ✅ Ссылка на Telegram бот работает

## 📮 Шаг 2: Массовая рассылка

**⚠️ ТОЛЬКО после успешного теста!**

```bash
npm run send-mass-email
```

**Что произойдет:**
- Загрузит ВСЕ записи из Firebase
- Покажет статистику и первых 5 получателей
- **Попросит подтверждение** (введите `yes`)
- Отправит письма всем (с задержкой 2 сек между отправками)
- Покажет итоговый отчет

## 🎨 Содержимое письма

Каждое письмо содержит:
- ❄️ Красивую шапку с градиентом
- 👋 Приветствие: "Здравствуйте, {firstName}!"
- 🎫 Номер регистрации (крупно и заметно)
- 🤖 Информацию о Telegram боте @ArctolympBot
- 📱 Большую кнопку для перехода в бот
- ✅ Список возможностей бота
- 💡 Полезные советы
- 🌟 Пожелания успеха

## ⚠️ Важно

1. **ВСЕГДА** делайте тест перед массовой рассылкой
2. Массовая рассылка требует **ручного подтверждения**
3. Между письмами задержка **2 секунды** (в повторной рассылке - **5 секунд**)
4. Время выполнения: **~30 писем в минуту** (в повторной - **~12 писем в минуту**)
5. Записи без `email` или `participantId` **пропускаются автоматически**

## 🔄 Повторная отправка

Если при массовой рассылке возникли ошибки **"Too many messages"** (лимит SMTP сервера):

```bash
# Отправка всем участникам (начиная с начала)
npm run retry-failed-emails

# Отправка начиная с определенной записи (например, с 500-й)
node src/scripts/retryFailedEmails.cjs 500
```

**Особенности повторной рассылки:**
- ✅ Увеличенная задержка между письмами (5 секунд вместо 2)
- ✅ Можно указать стартовый индекс (например, начать с 500-й записи)
- ✅ Отправляет письма **всем** участникам из базы (или начиная с указанного индекса)
- ✅ Если письмо уже было отправлено - получатель получит дубликат
- ✅ Сохраняет детальный отчет в файл `retry-email-report.json`
- ✅ Показывает реальный номер записи в базе данных

**Пример использования:**
```bash
# Начать рассылку с 500-й записи
node src/scripts/retryFailedEmails.cjs 500

# Начать рассылку с 1000-й записи
node src/scripts/retryFailedEmails.cjs 1000
```

## 🔧 Если что-то не работает

### Ошибка SMTP:
- Проверьте настройки в `.env`
- Используйте **пароль приложения**, не обычный пароль

### Нет участников:
- Проверьте Firebase коллекцию `registrations`
- Убедитесь, что есть поля `email` и `participantId`

### Письма не приходят:
- Проверьте папку "Спам"
- Проверьте лимиты SMTP-провайдера

## 📚 Рассылка по предметам

Для рассылки только участникам определенного предмета:

```bash
# Рассылка всем участникам с предметом "biology"
node src/scripts/sendEmailBySubject.cjs biology

# Рассылка участникам с предметом "biology", начиная с 500-й записи
node src/scripts/sendEmailBySubject.cjs biology 500

# Другие предметы:
node src/scripts/sendEmailBySubject.cjs math
node src/scripts/sendEmailBySubject.cjs physics
node src/scripts/sendEmailBySubject.cjs chemistry
```

**Особенности:**
- ✅ Фильтрует участников по предмету в поле `subjects`
- ✅ Поддерживает стартовый индекс (как в повторной рассылке)
- ✅ Показывает список предметов каждого участника
- ✅ Сохраняет отчет с указанием предмета

**Пример вывода:**
```
📧 РАССЫЛКА EMAIL ПО ПРЕДМЕТУ
✓ Предмет для рассылки: biology
📊 Всего участников с предметом "biology": 1250
📊 Участников к отправке: 1250

📋 ПЕРВЫЕ ПОЛУЧАТЕЛИ:
   1. Имя Фамилия - email@example.com (APO-2026-00001)
      Предметы: biology, math
   ...
```
